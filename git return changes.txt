ДО КОМІТІВ:
	git checkout -- NAME - відмінити зміну файла на вид який був до ваших змін
		..checkout . - відмінити всі зміни всіх файлів на вид...
	git clean - видалити новостворені файли
		ФЛАГИ(можна декілька разом): 
		-x - ігнорувати правила gitignore
		-d - удалити директорії які невідомі гіту(не були ще в комітах і пушнуті)
		-f - один з обов'язкових флагів, без нього нічо не буде

ДО КОМІТІВ в індексі(перенесли в поле для коміту/git add):
	або в гіт гуі просто перенести назад і використати попередні способи або
	git reset -- NAME - перенести назад, далі попередні команди

ПІСЛЯ КОМІТІВ:
	git commit --amend -m "комент коміту" - якщо ми закомітили і забули додати щось, 
		додаємо і далі ця команда оновить старий файл
	в git gui можна зробити все те саме кнопкою аменд біля поля вводу
	git reset HEAD(^ / ~num) - стрілка вверх - скільки комітів відмінити, або ~цифра - також скільки відмінити
		флаги: --soft - вернути до індексу(після add до коміту)
			--mixed - вернути до файлової системи(до add, тобто буде як абичний файл)
			--hard - видалити коміт повністю, вернути файлову систему до останнього коміту(видаляє
				нові файли, забирає те що додали у старих, до останнього коміту)
ПІСЛЯ ПУШУ:
	git revert LOG ID - відкат змін вибраного комміту, на гіті відображається як revert commit.
		після вводу відкривається вікно, нажимаєм insert, вводим текст ревкоміту, далі ентер і пишем wq
